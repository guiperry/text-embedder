G-Text Embedder

model landmark-lattice-v1  ·  768-dim int32 embeddings  ·  deterministic & zero-dependency

What it is

G-Text Embedder converts text into 768-dimensional deterministic integer vectors using the Landmark Lattice architecture — a pure feature-hashing algorithm with no ML models, no GPUs, and no external dependencies. The same input always produces the same output, across every CPU architecture and every software version. Model ID: landmark-lattice-v1.

Endpoints

MethodPathDescription
POST/embedEmbed a single text
POST/embed/batchEmbed up to 128 texts
POST/similarityCosine similarity between two texts
GET/healthLiveness check with model metadata

POST /embed

Request:

{ "text": "your text here" }

Response:

{
  "embedding": [942, 104, 3922, ...],
  "dimensions": 768,
  "model": "landmark-lattice-v1",
  "token_count": 4
}

POST /embed/batch

Request:

{ "texts": ["first text", "second text", "third text"] }

Response:

{
  "results": [
    { "index": 0, "embedding": [942, ...], "dimensions": 768, "token_count": 2 },
    { "index": 1, "embedding": [514, ...], "dimensions": 768, "token_count": 2 }
  ],
  "model": "landmark-lattice-v1",
  "count": 2
}

POST /similarity

Request:

{ "text_a": "first text", "text_b": "second text" }

Response:

{ "similarity": 0.7737, "model": "landmark-lattice-v1" }

Response field glossary

FieldTypeMeaning
embedding[]int32 768-element fixed-point vector, each element in [0, 10000]
dimensionsint Always 768
token_countint Number of tokens the input was split into (words + subword features)
modelstring Model identifier: landmark-lattice-v1
similarityfloat64 Cosine similarity in [0, 1] — how related two texts are

Curl examples

# Single embed
curl -X POST https://your-deployment.vercel.app/embed \
  -H "Content-Type: application/json" \
  -d '{"text":"the quick brown fox"}'

# Batch embed
curl -X POST https://your-deployment.vercel.app/embed/batch \
  -H "Content-Type: application/json" \
  -d '{"texts":["hello","world"]}'

# Similarity
curl -X POST https://your-deployment.vercel.app/similarity \
  -H "Content-Type: application/json" \
  -d '{"text_a":"machine learning","text_b":"deep learning"}'

# Health check
curl https://your-deployment.vercel.app/health

Go import usage

Use the embedding algorithm directly in your Go programs:

import "github.com/guiperry/text-embedder/pkg/embed"

// Embed returns a 768-element []int32 vector
vec := embed.Embed("the quick brown fox")

// Compare two texts
sim := embed.CosineSimilarity(
    embed.Embed("neural networks"),
    embed.Embed("deep learning"),
)
